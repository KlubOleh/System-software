Опис роботи програми
Метою даної курсової роботи є розробка спеціалізованого програмного забезпечення на мові C++, яке дозволить користувачу в реальному часі здійснювати моніторинг показників відеокарти та корегувати її робочі параметри (частоту, ліміти потужності, оберти вентиляторів) для досягнення оптимального балансу між продуктивністю та нагрівом.
2. Детальний опис функціональних модулів утиліти
Програма будується за модульним принципом, що забезпечує стабільність та легкість масштабування:
2.1. Модуль низькорівневої взаємодії (Hardware Abstraction Layer)Цей модуль є посередником між операційною системою Windows та драйвером відеокарти.
Використання NVAPI (NVIDIA SDK):Програма ініціалізує бібліотеку nvapi.dll, яка надає прямий доступ до внутрішніх регістрів GPU.
Функції ідентифікації: Визначення ідентифікатора пристрою (Device ID), версії мікропрограми (VBIOS) та поточної версії драйвера.
Зчитування сенсорів: Отримання даних з термодатчиків ядра, датчиків швидкості обертання вентиляторів (RPM) та лічильників енергоспоживання (TDP).
2.2. Модуль обробки телеметрії (Telemetry Engine)Модуль працює у фоновому потоці та виконує опитування заліза з частотою 1000 мс.
Моніторинг частот: Відстеження динамічної зміни частоти ядра (Boost Clock) під час навантаження в іграх (наприклад, PUBG).
Аналіз використання пам'яті: Контроль заповнення відеопам'яті (VRAM Usage) та шини передачі даних.
Логування: Можливість запису всіх отриманих даних у форматі .csv або .log для подальшого аналізу стабільності системи під навантаженням.
2.3. Модуль керування параметрами (Control & Optimization)Це активна частина програми, що дозволяє змінювати стан заліза:
Power Limit Scaling: Регулювання максимального TDP. Зниження цього показника на 10-20% дозволяє суттєво зменшити температуру без значної втрати FPS.
Custom Fan Curve: Користувач може самостійно задати графік залежності швидкості вентиляторів від температури. Наприклад: $50^\circ C \rightarrow 30\%$, $75^\circ C \rightarrow 80\%$, $85^\circ C \rightarrow 100\%$.
Profile Manager: Система збереження пресетів (Silent, Gaming, Extreme) для швидкого перемикання між ними.
3. Технічні вимоги до програмного забезпечення
3.1. Вимоги до апаратного та програмного середовищаОС: Microsoft Windows 10 або Windows 11 (64-bit).
Середовище розробки: Microsoft Visual Studio 2022 (MSVC compiler).Бібліотеки: NVIDIA NVAPI SDK, Qt Framework (для GUI) або ImGui.
Права доступу: Програма повинна запускатися з правами адміністратора для отримання доступу до низькорівневих викликів драйвера.
3.2. Нефункціональні вимоги (Якість та Надійність)
Продуктивність: Утиліта не повинна споживати більше 1% ресурсів CPU, щоб не створювати перешкод основному ігровому процесу.
Безпека: Програма повинна мати "запобіжники" — обмеження на введення занадто високих напруг або занадто низьких обертів вентиляторів при високих температурах.
Інтерфейс: Графічний інтерфейс має бути адаптивним (Dark Mode) та інформативним, з можливістю згортання в системний трей.
4. Алгоритм роботи програми
(Опис логіки)Процес роботи утиліти можна представити наступним циклом:
Ініціалізація: Завантаження драйверів, перевірка наявності сумісного GPU.
Основний цикл (Main Loop):Запит актуальних даних через NvAPI_GPU_GetThermalSettings.Отримання поточної частоти через NvAPI_GPU_GetAllClockFrequencies.Порівняння отриманої температури з критичною межею ($90^\circ C$).Якщо активовано автоматичний режим — коригування обертів кулерів згідно з обраною кривою.Оновлення віджетів інтерфейсу.
Завершення: При виході з програми — повернення параметрів відеокарти до заводських значень (для безпеки).
5. Сценарії використання та очікуваний результатВ результаті розробки студент отримує повноцінний інструмент, який може використовуватися у наступних випадках:
Діагностика: Перевірка системи охолодження ноутбука на предмет забруднення (якщо температури вищі за норму при тому ж навантаженні).
Оптимізація (Undervolting/Power Limiting): Налаштування відеокарти на роботу з меншою напругою, що зменшує шум і нагрів ноутбука Acer Nitro V15 під час тривалих ігрових сесій.
Автоматизація: Програма самостійно «підхоплює» потрібні налаштування при старті системи.
